# AI HR 智能招聘系统 — 方案说明

## 一、项目概述

AI HR 是一个基于 Next.js 14 的前端智能招聘管理系统原型，面向中小企业 HR 团队，覆盖从岗位发布、简历筛选、面试管理到录用决策的完整招聘链路。项目采用纯前端 + Mock 数据架构，可快速演示产品形态，后续可无缝对接真实后端 API。

**技术栈**：Next.js 14（App Router）+ React 18 + TypeScript + Tailwind CSS + 静态导出（`output: 'export'`）

**部署地址**：https://xiaoshan12138-ai.github.io/ai-hr/

---

## 二、问题诊断

项目初版存在三个核心问题：

1. **AI/HR 建议同质化**：所有候选人的 AI 建议和 HR 建议完全相同，未结合评分、缺口技能、薪资预期等个人数据生成差异化内容，丧失了"智能"系统的价值。

2. **状态与判断不一致**：候选人详情页的"最终判断"仅由分数推导（如 ≥75 分即"推荐通过"），与候选人实际状态字段（如 `rejected`）冲突，导致界面出现"淘汰状态却显示推荐通过"的逻辑矛盾。

3. **岗位页源码损坏与功能缺失**：`jobs/page.tsx` 文件因编码问题导致内容乱码，无法编辑；且上传简历后分析结果千篇一律，缺少将分析候选人加入候选人列表的入口。

## 三、方案设计

### 3.1 架构决策

采用 Next.js 静态导出方案（`output: 'export'`），生成纯静态 HTML/JS/CSS，部署到 GitHub Pages 零成本运行。数据层使用 TypeScript Mock 数据，类型定义严格，后续替换为 API 调用仅需修改数据源层。

### 3.2 页面结构

| 页面 | 路径 | 核心功能 |
|------|------|----------|
| 仪表盘 | `/dashboard` | KPI 卡片、招聘漏斗、热门岗位、待办事项、最新动态 |
| 岗位管理 | `/jobs` | 岗位列表/筛选 → 岗位详情（候选人列表+评分+简历上传分析+添加候选人） |
| 候选人 | `/candidates` | 候选人列表/搜索/筛选 → 详情（筛选结论、面试题库、面试录音、AI 评估与决策） |
| 面试管理 | `/interviews` | 面试排期与评估 |
| 系统设置 | `/settings` | 权重配置与阈值设置 |

### 3.3 评分体系

采用五分制映射（原始分/20），分级标准：
- ≥4.5 优秀 → 推荐通过，建议推进终面
- ≥4.0 良好 → 建议二面验证
- ≥3.5 待定 → 需补充面试
- <3.0 淘汰 → 不推荐当前岗位

## 四、AI 工具选型与使用理由

本项目使用 **Codex CLI（OpenAI）** 作为核心 AI 开发工具，理由如下：

1. **全栈代码生成与修复**：Codex 具备完整的文件读写、终端执行、TypeScript 编译检查能力，可在单次会话中完成"诊断→重写→验证→部署"全流程。
2. **上下文感知**：通过读取类型定义（`types.ts`）和 Mock 数据（`mock.ts`），AI 生成的代码能严格匹配数据结构，避免类型错误。
3. **迭代修复能力**：面对源码损坏、括号不平衡等复杂问题，AI 可逐步定位（tsc 报错行号→逐行检查→精准修复），比人工排查效率更高。
4. **部署自动化**：AI 可直接配置 GitHub Actions workflow、basePath 适配、lock 文件生成等 DevOps 环节，实现"对话式部署"。

## 五、关键配置

### 5.1 Next.js 静态导出
```js
// next.config.js
const nextConfig = {
  output: 'export',
  images: { unoptimized: true },
  basePath: '/ai-hr',
  assetPrefix: '/ai-hr/',
};
```
`basePath` 确保 GitHub Pages 子路径下路由和资源加载正确。

### 5.2 路由兼容
`layout.tsx` 中 `usePathname()` 返回包含 basePath 的路径，需手动剥离前缀后再做导航匹配：
```ts
const rawPath = pathname.startsWith("/ai-hr") ? pathname.slice(5) || "/" : pathname;
const activeValue = "/" + (rawPath.split("/")[1] || "dashboard");
```

### 5.3 GitHub Actions CI/CD
```yaml
# .github/workflows/deploy.yml
node-version: "22"    # 升级至 Node 22 避免弃用警告
run: npm ci           # 依赖 package-lock.json
run: npm run build    # next build → out/ 目录
```

### 5.4 页面间跳转
岗位页点击候选人时，通过 `sessionStorage` 传递候选人 ID，候选人页加载时自动打开对应详情，实现跨页面无刷新导航。

## 七、效果评估

| 维度 | 修复前 | 修复后 |
|------|--------|--------|
| AI 建议 | 所有人相同 | 基于分数/缺口/薪资/学历核验状态差异化输出 |
| 状态一致性 | 状态与判断矛盾 | finalDecision 严格映射 candidate.status |
| 岗位页简历分析 | 结果相同、无添加入口 | 按岗位+文件名生成差异化结果，支持一键添加候选人 |
| 候选人列表 | 岗位详情无候选人 | 展示评分+状态+点击跳转详情 |
| 构建稳定性 | 源码损坏无法编译 | tsc 零错误，括号全部平衡，GitHub Actions 自动部署 |
| 部署 | 无 | GitHub Pages 自动化部署，访问地址稳定可用 |
